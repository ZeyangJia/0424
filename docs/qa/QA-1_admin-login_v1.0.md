## 测试用例

### TC-001: 页面渲染

| 项目 | 内容 |
|------|------|
| 前置条件 | 应用已启动，访问 `http://localhost:5000` |
| 操作步骤 | 1. 打开浏览器访问 `/` |
| 预期结果 | 页面显示用户名输入框、密码输入框、登录按钮，无 JS 报错 |
| 优先级 | P0 |

### TC-002: 前端非空校验 — 用户名为空

| 项目 | 内容 |
|------|------|
| 前置条件 | 已打开登录页面 |
| 操作步骤 | 1. 留空用户名，填写密码<br>2. 点击登录按钮 |
| 预期结果 | 前端提示错误信息，不发送网络请求 |
| 优先级 | P0 |

### TC-003: 前端非空校验 — 密码为空

| 项目 | 内容 |
|------|------|
| 前置条件 | 已打开登录页面 |
| 操作步骤 | 1. 填写用户名，留空密码<br>2. 点击登录按钮 |
| 预期结果 | 前端提示错误信息，不发送网络请求 |
| 优先级 | P0 |

### TC-004: 登录成功

| 项目 | 内容 |
|------|------|
| 前置条件 | 已打开登录页面 |
| 操作步骤 | 1. 用户名输入 `admin`<br>2. 密码输入 `admin123`<br>3. 点击登录按钮 |
| 预期结果 | 登录成功，跳转至 `/admin`，页面展示 admin 后台内容 |
| 优先级 | P0 |

### TC-005: 登录失败 — 错误凭据

| 项目 | 内容 |
|------|------|
| 前置条件 | 已打开登录页面 |
| 操作步骤 | 1. 用户名输入 `admin`<br>2. 密码输入 `wrongpassword`<br>3. 点击登录按钮 |
| 预期结果 | 页面显示统一错误信息"用户名或密码错误"，不区分是用户名还是密码错误；不设置 session |
| 优先级 | P0 |

### TC-006: 登录失败 — 不存在的用户名

| 项目 | 内容 |
|------|------|
| 前置条件 | 已打开登录页面 |
| 操作步骤 | 1. 用户名输入 `notexist`<br>2. 密码输入 `admin123`<br>3. 点击登录按钮 |
| 预期结果 | 页面显示统一错误信息"用户名或密码错误" |
| 优先级 | P0 |

### TC-007: 未登录访问 /admin

| 项目 | 内容 |
|------|------|
| 前置条件 | 浏览器未持有有效 session |
| 操作步骤 | 1. 直接访问 `http://localhost:5000/admin` |
| 预期结果 | 重定向至登录页面 `/` |
| 优先级 | P0 |

### TC-008: 后端接口 — 空 JSON 请求

| 项目 | 内容 |
|------|------|
| 前置条件 | 应用已启动 |
| 操作步骤 | 1. POST `/api/login` with `{}` |
| 预期结果 | 返回 HTTP 401，`{ "success": false, "message": "用户名或密码错误" }` |
| 优先级 | P1 |

### TC-009: 后端接口 — 正确凭据

| 项目 | 内容 |
|------|------|
| 前置条件 | 应用已启动 |
| 操作步骤 | 1. POST `/api/login` with `{ "username": "admin", "password": "admin123" }` |
| 预期结果 | 返回 HTTP 200，`{ "success": true }`，Set-Cookie 包含 session |
| 优先级 | P0 |

## 环境

- Python 3.8+
- Flask 2.x+
- Chrome / Firefox / Edge（桌面端）

## 执行说明

1. 安装依赖：`pip install -r requirements.txt`
2. 启动应用：`python app.py`
3. 执行手动测试用例 TC-001 至 TC-007
4. 执行自动化测试：`python -m unittest tests/test_app.py`

## 缺陷

暂无

## 残留风险

| 风险 | 说明 |
|------|------|
| 硬编码凭据 | MVP 阶段使用，生产环境必须替换为数据库 + 哈希 |
| 无 HTTPS | 本地演示可用，生产必须启用 TLS |
